# 深圳技术大学足球协会系统 - 管理后台 (sztufa-admin)

本项目是校园足球信息管理平台的**后台管理端**，供协会管理员管理球队、球员信息以及录入比赛日程和实时关键事件。

## 🛠️ 技术栈
* **核心框架**：React 18 + TypeScript
* **构建工具**：Webpack 5
* **图片直传**：集成 Cloudflare R2 对象存储，上传图片在前端动态转换为 `FormData` 流上传，替换 Base64 格式
* **API 通信**：基于 `axios` 封装，支持根据访问的域名（`localhost` 或生产域名）自适应动态切换请求地址

## 🌟 核心功能
1. **用户认证**：登录后自动缓存 JWT Token，并通过拦截器自动附加到请求头。
2. **球队管理**：
   * 创建、更新球队基本信息。
   * 支持通过 R2 接口直传高分辨率队徽。
   * **数据统计与战绩走势**：展示本赛季累计零封场次及最近 5 比赛 W/D/L 胜平负圆形走势标。
3. **球员管理与状态同步**：
   * 关联特定球队，进行球员信息的增删改查。
   * 支持球员状态、黄红牌数的行内修改同步，自动对停赛球员（状态为 `suspended`）予以粉红行背景高亮和 `🛑 停赛` 徽章。
   * **批量导入与自适应优化 [NEW]**：支持批量添加球员时的分步保存进度条提示（毛玻璃悬浮遮罩）；全面优化球员与球队表格的移动端窄屏排版约束，支持横向滑动，防止因列收缩挤压变形。
4. **赛事事件实时管理**：
   * 管理比赛日程、地点、比分。
   * **比赛关键事件流录入**：支持普通进球 ⚽、点球 🎯、乌龙球 🥅、换人 🔄、黄牌 🟨、红牌 🟥。
   * **进球助攻录入**：当事件为“普通进球”或“点球”时，提供本方球队的其他球员列表以录入“助攻球员”。
   * **智能业务规则强校验**：
     * 换人事件：联动“换上球员”和“换下球员”双向列表，校验防重，已被换下的球员不可二次换上。
     * 得分联动：主队得分 = 主队（普通进球 + 点球）+ 客队乌龙球。自动校验得分一致性，防止误录。
   * **停赛预警机制**：比分录入的球员下拉选择器中，已被停赛的球员文字被自动标红，且带有 `(🛑 停赛 - 🟨X 🟥Y)` 后缀。
   * **全场最佳评选 (MVP)**：支持在比赛基本信息编辑区，跨主客队直接评选并保存该场最佳球员（MVP）荣誉。
5. **系统操作审计日志 [NEW]**：
   * 独立的审计日志面板，支持按操作人员与操作类型进行多维度精准筛选过滤。
   * 支持短时连续批量操作的智能聚合展示，对具体字段的修改差值细节（如比分变动、球员属性、账号权限等）实现折叠与展开收起交互。
   * 移动端自适应卡片化排版，且长描述文本自动折行对齐，彻底优化移动端下的操作与浏览体验。
6. **云存储安全备份与还原**：
   * 支持一键手动触发全库数据备份推送至 Cloudflare R2。
   * 列表化下载和查看云端历史备份文件，并提供一键高一致性事务级整库“覆盖还原”数据库。
7. **赛季一键归档与重置 [NEW]**：
   * 新增赛季管理卡片，管理员可以一键归档当前赛季数据并指定开启新赛季（如“2026秋季赛季”）。
   * 归档时自动触发数据库级事务，重置所有球员的红黄牌数与停赛标志为可用（`active`）。

## 🚀 快速启动

### 1. 安装依赖
```bash
npm install
```

### 2. 本地开发调试
```bash
npm start
```
开发服务器默认在 `http://localhost:3002` 启动，并会自动代理 `/api` 请求至本地后端 (127.0.0.1:3001)。

### 3. 生产构建
```bash
npm run build
```
编译产物会输出至 `build/` 目录。
